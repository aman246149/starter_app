# Feature Template

Template structure for creating new features following the feature-first architecture.

## Complete Feature Structure

```text
lib/features/[feature_name]/
├── domain/                          # Business logic layer
│   ├── entities/                    # Core business objects
│   │   └── [feature]_entity.dart
│   ├── value_objects/               # Domain values without identity
│   │   └── [feature]_value_object.dart
│   └── repositories/                # Repository interfaces (ports)
│       └── i_[feature]_repository.dart
│
├── application/                     # Use cases layer (CQRS pattern)
│   └── usecases/                    # Business workflows
│       ├── get_[feature].dart       # Query (read operation)
│       ├── create_[feature].dart    # Command (write operation)
│       └── update_[feature].dart    # Command (write operation)
│
├── infrastructure/                  # Data layer
│   ├── datasources/                 # API & database implementations
│   │   ├── [feature]_remote_data_source.dart
│   │   └── [feature]_local_data_source.dart
│   ├── models/                      # DTOs with serialization
│   │   └── [feature]_model.dart
│   └── repositories/                # Repository implementations
│       └── [feature]_repository_impl.dart
│
├── l10n/                            # Feature-specific localization
│   ├── [feature]_en.arb             # English strings
│   ├── [feature]_es.arb             # Spanish strings (add more as needed)
│   ├── l10n_extensions.dart         # Context extension (e.g., context.authL10n)
│   └── [feature]_localizations.dart # Generated by gen_l10n
│
├── presentation/                    # UI layer
│   ├── bloc/                        # State management
│   │   ├── [feature]_bloc.dart
│   │   ├── [feature]_event.dart
│   │   └── [feature]_state.dart
│   ├── failure_message/             # Failure to UI message mappers
│   │   └── [feature]_failure_mapper.dart
│   ├── pages/                       # Top-level screens
│   │   └── [feature]_page.dart
│   ├── widgets/                     # Feature-specific widgets
│   │   └── [feature]_widget.dart
│   └── routes/                      # Route declarations
│       └── [feature]_routes.dart    # part of app_router.dart
│
└── [feature_name].dart              # Barrel export file
```

## Route Declaration Pattern

Every feature with navigation should have a routes file:

```dart
// lib/features/[feature]/presentation/routes/[feature]_routes.dart
part of '../../../../core/navigation/app_router.dart';

/// Main route for this feature
@TypedGoRoute<FeatureRoute>(
  path: RouteDefinitions.featurePath,
  name: RouteDefinitions.featureName,
)
class FeatureRoute extends BaseRoute with $FeatureRoute {
  const FeatureRoute();
  
  @override
  Widget build(BuildContext context, GoRouterState state) {
    return const FeaturePage();
  }
}

/// Detail route (optional)
@TypedGoRoute<FeatureDetailRoute>(
  path: '${RouteDefinitions.featurePath}/:id',
  name: 'feature-detail',
)
class FeatureDetailRoute extends BaseRoute with $FeatureDetailRoute {
  const FeatureDetailRoute(this.id);
  final String id;
  
  @override
  Widget build(BuildContext context, GoRouterState state) {
    return FeatureDetailPage(id: id);
  }
}
```

## Barrel Export Pattern

```dart
// lib/features/[feature]/[feature].dart
/// [Feature] module.
///
/// Description of what this feature does.
library;

// Presentation
export 'presentation/pages/[feature]_page.dart';

// Future exports (uncomment when implemented):
// Domain
// export 'domain/entities/[feature]_entity.dart';
// export 'domain/repositories/i_[feature]_repository.dart';

// Application
// export 'application/usecases/get_[feature].dart';
```

## Creating a New Feature: Step-by-Step

### 1. Create Directory Structure

```bash
mkdir -p lib/features/my_feature/presentation/{pages,routes}
```

### 2. Create Page

```dart
// lib/features/my_feature/presentation/pages/my_feature_page.dart
import 'package:flutter/material.dart';

class MyFeaturePage extends StatelessWidget {
  const MyFeaturePage({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('My Feature')),
      body: const Center(child: Text('My Feature - Coming Soon')),
    );
  }
}
```

### 3. Create Route

```dart
// lib/features/my_feature/presentation/routes/my_feature_routes.dart
part of '../../../../core/navigation/app_router.dart';

@TypedGoRoute<MyFeatureRoute>(
  path: RouteDefinitions.myFeaturePath,
  name: RouteDefinitions.myFeatureName,
)
class MyFeatureRoute extends BaseRoute with $MyFeatureRoute {
  const MyFeatureRoute();
  
  @override
  Widget build(BuildContext context, GoRouterState state) {
    return const MyFeaturePage();
  }
}
```

### 4. Create Barrel Export

```dart
// lib/features/my_feature/my_feature.dart
library;

export 'presentation/pages/my_feature_page.dart';
```

### 5. Add Route Definition

```dart
// lib/core/navigation/route_definitions.dart
static const String myFeaturePath = '/my-feature';
static const String myFeatureName = 'my-feature';
```

### 6. Register in AppRouter

```dart
// lib/core/navigation/app_router.dart
import 'package:starter_app/features/my_feature/presentation/pages/my_feature_page.dart';

part '../../features/my_feature/presentation/routes/my_feature_routes.dart';
```

### 7. Generate Code

```bash
dart run build_runner build --delete-conflicting-outputs
```

### 8. Navigate

```dart
const MyFeatureRoute().go(context);
```

## Best Practices

### DO ✅

- Keep routes with their feature in `presentation/routes/`
- Use `part of` to include routes in main router
- Follow the layer structure (domain → application → infrastructure → presentation)
- Export only necessary classes in barrel file
- Document each layer's purpose
- Use route definitions for all paths

### DON'T ❌

- Don't put routes in `core/navigation/` (that's only for infrastructure)
- Don't skip layers (every feature should have at least presentation)
- Don't let features depend on each other directly
- Don't mix feature logic in core modules
- Don't hardcode paths in route files
